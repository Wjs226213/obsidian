# Linux 自动化学习路线

> 适用基础：Python ✓ | Linux 基础 ✓ | MySQL 基础 ✓
> 目标：从手动运维 → 脚本自动化 → 基础设施即代码（IaC）→ AIOps

---

## 一、学习路线总览

```
阶段一：Shell 自动化（基础）
   ↓
阶段二：Python 自动化运维（进阶）
   ↓
阶段三：配置管理与 IaC（核心）
   ↓
阶段四：CI/CD 流水线（工程化）
   ↓
阶段五：容器化与编排（主流）
   ↓
阶段六：监控与告警（闭环）
   ↓
阶段七：AIOps（未来的方向）
```

---

## 二、各阶段详解

### 阶段一：Shell 自动化（基础）

**学习理由**：所有自动化脚本的起点，系统管理员的必修课。

| 知识点 | 掌握程度 | 说明 |
|--------|---------|------|
| 变量、条件判断、循环 | ★★★ 熟练 | Shell 编程基础 |
| 函数与作用域 | ★★★ 熟练 | 模块化脚本 |
| sed / awk 文本处理 | ★★★ 熟练 | 日志解析、配置修改 |
| find / xargs / grep 组合 | ★★★ 熟练 | 文件批量操作 |
| 正则表达式 | ★★★ 熟练 | 文本匹配核心技能 |
| crontab / systemd timer | ★★★ 熟练 | 定时任务 |

**练习项目**：
- 日志轮转脚本（按日期压缩、清理过期日志）
- 批量用户管理脚本（创建/删除/权限设置）
- 系统健康检查脚本（CPU/内存/磁盘 → 生成 HTML 报告）

---

### 阶段二：Python 自动化运维（进阶）

**学习理由**：你已经会 Python，用它写自动化比 Shell 更强大（跨平台、网络请求、数据处理）。

| 知识点 | 掌握程度 | 说明 |
|--------|---------|------|
| `os` / `sys` / `subprocess` | ★★★ 熟练 | 系统命令调用 |
| `shutil` / `glob` / `pathlib` | ★★★ 熟练 | 文件批量操作 |
| `paramiko` | ★★★ 熟练 | SSH 远程执行命令 |
| `fabric` | ★★☆ 掌握 | 简化远程部署 |
| `psutil` | ★★☆ 掌握 | 系统监控（CPU/内存/进程） |
| `watchdog` | ★☆☆ 了解 | 文件系统事件监控 |

**练习项目**：
- 批量服务器巡检工具（SSH 连接 → 采集信息 → 写入数据库）
- 日志分析告警系统（Tail 日志 → 正则匹配 → 钉钉/邮件通知）
- 文件同步与备份工具（增量备份 + 版本控制）

---

### 阶段三：配置管理与 IaC（核心）

**学习理由**：这是自动化运维的**核心技能**，让基础设施像代码一样管理。

#### Ansible（首选推荐）

```
你已有 Python + SSH 基础 → Ansible 零成本上手
Agentless（无需在目标机器装客户端）→ 最易入门
```

| 知识点 | 掌握程度 | 说明 |
|--------|---------|------|
| inventory 主机清单 | ★★★ 熟练 | 管理服务器列表 |
| Playbook 编写 | ★★★ 熟练 | YAML 格式的自动化剧本 |
| Module 使用 | ★★★ 熟练 | shell/copy/template/service 等 |
| Role 角色封装 | ★★★ 熟练 | 可复用的自动化单元 |
| Ansible Vault | ★★☆ 掌握 | 敏感信息加密 |
| Jinja2 模板 | ★★☆ 掌握 | 动态配置文件生成 |

#### Terraform（IaC 标准）

| 知识点 | 掌握程度 | 说明 |
|--------|---------|------|
| HCL 语法 | ★★☆ 掌握 | Terraform 配置语言 |
| Provider 管理 | ★★☆ 掌握 | 云平台/服务商对接 |
| State 管理 | ★★☆ 掌握 | 状态文件管理 |
| Module 封装 | ★★☆ 掌握 | 可复用的基础设施组件 |

---

### 阶段四：CI/CD 流水线（工程化）

**学习理由**：自动化不只是运维，还有代码的构建、测试、部署全流程。

| 工具 | 掌握程度 | 说明 |
|------|---------|------|
| **Git** | ★★★ 熟练 | 分支策略、Git Flow |
| **GitHub Actions** | ★★★ 熟练 | CI/CD 流程编排 |
| **GitLab CI** | ★★★ 熟练 | .gitlab-ci.yml 编写 |
| **Jenkins** | ★★☆ 掌握 | Pipeline as Code |
| **ArgoCD** | ★☆☆ 了解 | GitOps 持续部署 |

**练习项目**：
- 搭建 Flask 项目的完整 CI/CD 流水线
- 代码提交 → 自动测试 → 自动部署到服务器
- 加上你的博客项目：push → lint → test → deploy

---

### 阶段五：容器化与编排（主流）

**学习理由**：当前业界标准，几乎所有应用都跑在容器里。

| 知识点 | 掌握程度 | 说明 |
|--------|---------|------|
| Docker 基础命令 | ★★★ 熟练 | build/run/exec/compose |
| Dockerfile 编写 | ★★★ 熟练 | 多阶段构建、镜像优化 |
| Docker Compose | ★★★ 熟练 | 多容器编排 |
| K8s 核心概念 | ★★☆ 掌握 | Pod/Service/Deployment/ConfigMap |
| k3s / minikube | ★★☆ 掌握 | 本地 K8s 环境 |

**练习项目**：
- 将你的 Flask 博客项目 Docker 化
- 用 Docker Compose 编排 Flask + SQLite + Nginx
- 在 K3s 上部署你的应用

---

### 阶段六：监控与告警（闭环）

**学习理由**：没有监控的自动化是盲人摸象。

| 知识点 | 掌握程度 | 说明 |
|--------|---------|------|
| Prometheus 架构 | ★★☆ 掌握 | 指标采集与存储 |
| Node Exporter | ★★☆ 掌握 | 系统指标采集 |
| Grafana 可视化 | ★★☆ 掌握 | 仪表盘制作 |
| AlertManager | ★★☆ 掌握 | 告警规则与通知 |
| ELK / Loki | ★☆☆ 了解 | 日志集中管理 |

**练习项目**：
- 监控你的 Flask 博客（请求量/响应时间/错误率）
- 配置告警规则（CPU > 80% → 钉钉通知）
- 搭建 Loki + Grafana 日志可视化

---

### 阶段七：AIOps（未来方向）

**学习理由**：你有深度学习基础，这是你的差异化优势。

| 知识点 | 掌握程度 | 说明 |
|--------|---------|------|
| 时序异常检测 | ★★☆ 掌握 | LSTM/Transformer 检测指标异常 |
| 日志聚类分析 | ★★☆ 掌握 | 自动归类错误日志 |
| RAG + 运维知识库 | ★★☆ 掌握 | 故障排查 Agent |
| LLM + 自动化 | ★★☆ 掌握 | 自然语言驱动运维操作 |

## 四、给你的建议学习顺序

```
第 1-2 周：Shell 脚本进阶（你已有基础，查漏补缺）
第 3-4 周：Python 自动化脚本（paramiko + psutil）
第 5-8 周：Ansible（从 Playbook 到 Role）
第 9-10 周：Docker + Compose（容器化你的博客）
第 11-12 周：CI/CD（GitHub Actions 自动部署）
第 13-14 周：Prometheus + Grafana（监控体系）
第 15-16 周：K8s 入门（可选）
后续：AIOps 探索（你的优势方向）
```

---

## 五、一句话建议

> **你已经有 Python + Linux 基础，直接跳进 Ansible 和 Docker 性价比最高，边学边把自己的项目自动化起来。**
